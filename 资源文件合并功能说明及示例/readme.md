### 资源文件合并功能说明及示例

#### 说明
  * 此脚本用于合并多个不同文件为一个文件
  * 主要用于如嵌入式开发中，将多个不同长度的文件合并为一个带索引头的bin文件。
  * 使用脚本方式加载文件，以用于后继增减文件方便，脚本文件格式见例“MargeScript.txt”
  * 脚本文件中指定了起始存放位置，以用于后继使用UBIN或本软件进一步合并入应用。
  
#### 目标文件并按下述方式组织：
  目标文件以“数据头”+被合并文件数据流阵列“方式组织，
#### 数据头
  * 数据头长度：如脚本文件中指定最终文件为7个，则数据头长度为(7+1)*4(默认)=32个,最后多存一位，用于最后一个被合并文件数据计算长度。
  * 数据头格式：4字节(默认，可在配置中修改)为单位，高端方式排列(高位在前，可在配置中修改为LSB)
  * 被合并文件数据流阵列: 紧跟数据后，每个被合并文件数据之间没有间隔
  * 被合并文件数据流阵列大小：= 下个数据头值 - 当前数据头值
  
#### 生成示例
  * 示例脚本：MargeScript.txt
  * 示例源文件：B0.bmp B1.bmp
  * 示例生成的目标文件： 生成的目标文件.bin
##### 脚本分析：
  * 脚本文件中指定最终文件为7个
  * 脚本索引对齐方式为：0即MSB（高位在前）
  * 脚本文件中指定基址为0x00004000
  * 第1个文件长度为40字节, 第2个文件先预留，第3个文件长度为60字节，后续5个文件也预留
  * 上述描述后，目标文件大小为： 数据头8*4=32个 +第1文件40 + 第3文件60 = 132个字节
  * 生成文件"生成的目标文件.bin" 格式数据头为：
    + 0x00 0x00 0x40 0x20  0x00 0x00 0x40 0x48  0x00 0x00 0x40 0x48  0x00 0x00 0x40 0x84
    + 0x00 0x00 0x40 0x84  0x00 0x00 0x40 0x84  0x00 0x00 0x40 0x84  0x00 0x00 0x40 0x84
##### 解析
  * 第1个文件数据最终从0x00004020开始:
    + 存放在“0x00004020 - 基址0x00004000” = 0x20位置
    + 长度为2个文件数据 “0x00004048 - 基址0x00004020” = 0x28,即40字节
  * 第2个文件数据最终从0x00004048开始:
    + 存放在“0x00004048 - 基址0x00004000” = 0x48位置
    + 长度为2个文件数据 “0x00004048 - 基址0x00004048” = 0,即此文件为空
  * ... 后续以此方式进行
